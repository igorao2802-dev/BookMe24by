# 🏥 bookme24.by — Система онлайн-записи для салона красоты

**BookMe24** — это современная система онлайн-записи для салонов красоты «Здоровье и красота». Приложение позволяет клиентам записываться на услуги 24/7, а администраторам — эффективно управлять расписанием, услугами и специалистами.

## 🌟 Особенности

### Для клиентов

- ✅ **Онлайн-запись** — запишитесь за 5 простых шагов (менее 2 минут)
- ✅ **Каталог услуг** — подробное описание услуг с фильтрацией и сортировкой
- ✅ **Выбор специалиста** — просмотр профилей мастеров и их услуг
- ✅ **Личный кабинет** — управление своими записями, история посещений
- ✅ **Избранное** — быстрый доступ к любимым услугам и мастерам
- ✅ **Бонусная система** — накопление баллов за посещения (в процессе разработки)
- ✅ **Две языковые версии** — русский и английский языки (в процессе доработки)
- ✅ **Тёмная/светлая тема** — комфорт в любое время суток

### Для специалистов

- ✅ **Расписание** — просмотр своих записей на выбранную дату
- ✅ **Профиль мастера** — услуги, статистика, KPI
- ✅ **Переключение роли** — демо-режим «Клиент / Специалист / Админ» через `RoleSwitcher`

### Для администраторов

- ✅ **Управление записями** — просмотр, редактирование, отмена записей
- ✅ **Управление услугами** — добавление, редактирование, удаление услуг
- ✅ **Управление специалистами** — назначение услуг мастерам
- ✅ **Статистика** — отслеживание выручки, количества записей
- ✅ **Фильтры и поиск** — быстрый поиск по клиентам, датам, статусам
- ✅ **Валидация данных** — защита от некорректного ввода

## 🚀 Технологии

### Frontend

- **React 18.3** — UI-библиотека
- **React Router v6** — клиентский роутинг
- **Context API** — тема и язык (`ThemeContext`, `LanguageContext`)
- **Custom Hooks** — бизнес-логика вынесена в `src/hooks/`
- **CSS3** — BEM, дизайн-токены в `tokens.css`
- **Lucide React** — иконки

### Зависимости (production)

- **date-fns** — работа с датами
- **react-datepicker** — выбор даты в форме записи
- **react-hot-toast** — уведомления (обёртка `Toast.jsx` в UI-слое)
- **nanoid** — генерация ID
- **lucide-react** — иконки

> Ранее рассматривались `react-hook-form`, `@hookform/resolvers`, `zod` — **удалены**; формы и валидация реализованы на собственных хуках и `validators.js`.

### Хранение данных

- **localStorage** — сохранение данных на клиенте
- **JSON-файлы** — начальный каталог услуг и специалистов

### Инструменты

- **Create React App** — сборка и dev-сервер
- **ESLint** — конфиг `react-app` (запускается при `npm test` и `npm run build`, отдельного скрипта `lint` нет)
- **Jest + Testing Library** — unit-тесты (`npm test`)

### npm-скрипты

| Команда                | Назначение                                                             |
| ---------------------- | ---------------------------------------------------------------------- |
| `npm start`            | Dev-сервер на [http://localhost:3000](http://localhost:3000)           |
| `npm test`             | Unit-тесты (Jest, интерактивный режим)                                 |
| `npm run build`        | Production-сборка в `build/`                                           |
| `npm run analyze`      | Анализ размера бандла (после `build`; требует `source-map-explorer`)   |

## 📦 Установка

### Требования

- Node.js >= 18.x
- npm >= 8.x

### Шаги установки

1. **Клонируйте репозиторий:**

```bash
git clone https://github.com/igorao2802-dev/BookMe24by.git
cd BookMe24by
```

2. **Установите зависимости:**

```bash
npm install
```

3. **Запустите проект:**

```bash
npm start
```

Приложение откроется по адресу [http://localhost:3000](http://localhost:3000)

4. **Сборка для production:**

```bash
npm run build
```

### Архитектурные решения

- **Дизайн-токены:** палитра light/dark — единый источник в `src/styles/tokens.css`.
- **Навигация и роли:** `src/config/navigation.js` + `src/utils/roleRouting.js`; маршруты — `AppRoutes.jsx`.
- **Бизнес-логика:** многошаговая запись — `useBookingWizard.js` (+ подмодули `hooks/bookingWizard/`); админ-панель — `useAdminDashboard.js` (+ `hooks/adminDashboard/`).
- **Уведомления:** `react-hot-toast` и `Toast.jsx` в UI-слое; хуки CRUD возвращают `{ success, error, data }`, Toast вызывает UI через `createNotify.js`.

### Константы приложения

Основные константы находятся в `src/utils/constants.js`:

- **BOOKING_STATUS** — статусы записей (pending, confirmed, inProgress, completed, cancelled)
- **SERVICE_CATEGORIES** — категории услуг (hair, nails, massage, cosmetology, spa)
- **STORAGE_KEYS** — ключи для localStorage
- **BUSINESS_CONFIG** — бизнес-правила (время работы, буфер между записями и т.д.)
- **PRICE_LIMITS** — ограничения цен (min, max, step)
- **RATE_LIMITS** — ограничения для защиты от спама

## 📋 Функциональность

### Улучшения (доработки) после проверки

Выполнен рефакторинг кода с целью его оптимизации и улучшения поддержки:

- **useLocalStorage** — исправлен race condition при быстрых последовательных обновлениях; добавлен stress-тест
- **Toast** — устранено дублирование уведомлений: вынесен из бизнес-хуков в UI-слой
- **Валидация телефона** — согласована между `validators.js` и формой записи (формат РБ, коды операторов)
- **Комментарии** — удалены мусорные комментарии; полезные пояснения сохранены в формате «ПОЧЕМУ»
- **Мемоизация** — аудит `useMemo` / `useCallback` завершён: убрана избыточная, оставлена обоснованная

### Многошаговая запись (5 шагов)

1. **Выбор услуги** — поиск, фильтрация по категории, цене, рейтингу
2. **Выбор специалиста** — отображение только мастеров, оказывающих выбранную услугу
3. **Дата и время** — выбор свободных окон с учётом расписания мастера
4. **Контактные данные** — валидация телефона (РБ), email, комментария
5. **Подтверждение** — проверка всех данных перед созданием записи

### Каталог услуг и специалистов

**Фильтры:**

- Категория услуги (Все, Волосы, Ногти, Массаж, Косметология, SPA)
- Диапазон цен (от 0 до 500 BYN)
- Рейтинг (Любой, 4+, 4.5+, 4.8+)

**Сортировка:**

- По популярности
- По цене (возрастание/убывание)
- По рейтингу
- По алфавиту
- По категории

### Админ-панель

**Управление записями:**

- Просмотр всех записей с пагинацией
- Фильтрация по статусу, специалисту, дате
- Редактирование записей
- Отмена записей с подтверждением

**Управление услугами:**

- Добавление новых услуг
- Редактирование существующих
- Удаление (только кастомных услуг)
- Назначение специалистов на услуги
- Валидация полей (название, категория, описание, длительность, цена)

**Управление специалистами:**

- Добавление новых мастеров
- Редактирование данных
- Удаление (только кастомных)
- Назначение услуг специалистам

### Личный кабинет

**Разделы:**

- Профиль (ФИО, телефон, email)
- Статистика (всего записей, предстоящие, подтверждённые, отменённые)
- История записей (все, предстоящие, завершённые, отменённые)
- Избранное (услуги и специалисты)
- Настройки (телефон, email) - в процессе доработки
- Бонусный счёт (баланс, уровень, история начислений) - в проекте

## 🔒 Безопасность

### Защита от XSS

**Уровень 1: Блокировка на вводе:**

- Формы: `BookingForm`, `ServiceForm`, `SpecialistForm`
- Фильтрация в `onChange` через `sanitizeInput()` и специализированные фильтры (`sanitizeNameInput`, `sanitizePhoneInput`)
- Запрещённые символы: `< > " ' ; & | \ $`

**Уровень 2: Валидация:**

- Функция `hasXSSPattern()` в `validators.js`
- Проверка на спецсимволы и типичные XSS-паттерны (`<script>`, `onerror=`, `javascript:`)
- При обнаружении — ошибка `validation.xssDetected`

**Уровень 3: Очистка перед сохранением:**

- Функция `sanitizeObject()` в `storageHelper.js`
- Рекурсивная очистка всех строковых полей перед записью в `localStorage`
- `sanitizeInput()` — универсальный алиас `sanitizeString()` для форм и submit-хендлеров

**Рендеринг:**

- Только React JSX
- Без `dangerouslySetInnerHTML` и `innerHTML`

**Модули:**

- `src/utils/sanitizers.js`
- `src/utils/validators.js`
- `src/utils/storageHelper.js`

### Rate Limiting

Защита от многократных кликов:

- ≤ 3 клика за 5 секунд (короткое окно)
- ≤ 10 кликов за 30 секунд (длинное окно)
- Блокировка на 30 секунд при превышении лимита

### Валидация

**Телефон (РБ):**

- Формат: +375XXXXXXXXX (12 цифр после «+»)
- Разрешённые коды операторов: 17, 25, 29, 33, 44

**Email:**

- Валидация по RFC 5321
- Максимальная длина: 254 символа

**Цены:**

- Только целые числа
- Диапазон: 0 — 10000 BYN

**Длительность:**

- Минимум: 15 минут
- Максимум: 480 минут (8 часов)

## ⚠️ Ограничения

- **localStorage вместо БД** — учебный проект; данные хранятся в браузере клиента, без серверной синхронизации
- **Только unit-тесты** — полноценный E2E-набор не настроен; Playwright используется только в `scripts/stress-test-*.js`
- **Нет отдельного lint-скрипта** — ESLint запускается только через CRA при `test` / `build`
